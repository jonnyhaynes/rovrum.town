# Implementation plans

One markdown file per unit of work, named after its GitHub issue (e.g.
`issue-42-source-adapter-interface.md`) or a short slug when there's no issue yet.
Claude drafts the plan here; **a human approves it before any code is
written** (see `../dev-workflow.md`, step 2).

A plan is the thing that gets reviewed -- it's diffable, referenceable, and outlives
any single Claude session. Keep approved plans in the repo so the PR can reference
them and reviewers can check the diff against the plan that was agreed.
